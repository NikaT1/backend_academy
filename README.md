### Проект 2 - Консольное приложение для генерации лабиринтов и поиска пути в них

#### Запуск приложения

При запуске приложения могут быть переданы следующие параметры:
- `-m` (`-maze`) - выбор алгоритма генерации лабиринта (допустимые значения: PRIM, RECURSIVE_BACKTRACKER);
- `-p` (`-path`) - выбор алгоритма поиска пути в лабиринте (допустимые значения: DFS, A_STAR).

Данные параметры являются необязательными.

После запуска приложения пользователю будет предложено ввести следующие значения:
- *алгоритм генерации лабиринта* (если он не был задан через параметр при запуске);
- *алгоритм поиска пути в лабиринте* (если он не был задан через параметр при запуске);
- *ширина и высота лабиринта*;
- *координаты начальной и конечной точки* для поиска пути в лабиринте.
Для выбора случайного значения для параметра следует вводить пустую строку.

#### Отрисовка лабиринта
При отрисовке лабиринта используются следужеющие цветовые обозначения:
- *желтый* - границы лабиринта;
- *синий* - путь в лабиринте;
- *красный* - конечная точка пути;
- *зеленый* - стартовая точка пути.

#### Перечень используемых алгоритмов

Генерация лабиринта:
- алгоритм Прима (обозначение *PRIM*);
- алгоритм recursive backtracker (обозначение *RECURSIVE_BACKTRACKER*).

Поиск пути в лабиринте:
- поиск в глубину (обозначение *DFS*);
- алгоритм поиска А* (обозначение *A_STAR*).

### Проект 3 - Консольное приложение для анализа лог-файлов и генерации отчетов

#### Запуск приложения

При запуске приложения могут быть переданы следующие параметры:

- `-p` (`--path`) - путь к файлу/файлам с логами (URL или путь в файловой системе);
- `-f` (`--from`) - начальный временной параметр для анализа логов в формате ISO8601 (опциональный);
- `-t` (`--to`) - конечный временной параметр для анализа логов в формате ISO8601 (опциональный);
- `-F` (`--format`) - формат генерации отчета (опциональный; допустимые значения: ADOC, MARKDOWN; значение по-умолчанию
  MARKDOWN);
- `--filter-field` - поле, по которому будет производиться фильтрация (опциональный).
- `--filter-value` - регулярное выражение для фильтрации лог-записей (опциональный).

Параметры --filter-field и --filter-value применяются только совместно.

#### Генерация отчета

После запуска программы пользователю будет выведен текстовый отчет в выбранном формате (по умолчанию - MARKDOWN).

#### Допустимые значения filter-field

- REMOTE_ADDR
- REMOTE_USER
- TIME_LOCAL
- HTTP_METHOD
- RESOURCE
- HTTP_VERSION
- STATUS
- BODY_BYTES_SENT
- HTTP_REFERER
- HTTP_USER_AGENT

### Проект 4 - Фрактальное пламя

#### Параметры для запуска

- `-h`, `-height`: height of image
- `-w`, `-width`: width of image
- `-s`, `-samples`: number of samples
- `-i`, `-iterations`: iterations per sample
- `-S`, `-symmetry`: symmetry level
- `-g`, `-gamma`: gamma fot correction
- `-f`, `-funcs`: count of affine functions
- `-t`, `-threads`: count of threads
- `-min_step`: min count of steps before rendering a sample
- `-xmin`: min value of x
- `-xmax`: max value of x
- `-ymin`: min value of y
- `-ymax`: max value of y
- `-p`, `-path`: path of result file
- `-T`, `-transformations`: list of transformations,
- `-F`, `-format`: format of result file;

Все параметры запуска являются необязательными.

#### Доступные трансформации

- DIAMOND
- FISHEYE
- HEART
- LINEAR
- SINUSOIDAL
- SPHERICAL
- SWIRL

#### Доступные форматы изображения

- PNG
- JPEG
- BMP
